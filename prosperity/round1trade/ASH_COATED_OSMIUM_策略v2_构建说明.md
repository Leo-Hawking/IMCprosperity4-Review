# ASH_COATED_OSMIUM 策略 v2 构建说明

## 1. 市场前提

### 1.1 OU 模型参数

标的真实 fair price 服从 OU 过程：

| 参数 | 值 |
|---|---|
| μ（长期均值） | 10000（固定） |
| θ̂（drift 回归） | 0.00415 |
| θ_acf（ACF 估计） | 0.00423 |
| σ_ε（残差波动） | 0.4300 |
| σ_p（稳态波动，观测） | 4.6935 |
| σ_p（稳态波动，理论） | 4.7227 |
| σ_Δp（单步变化波动） | 0.4304 |

**时间尺度**：1 个时间步 = 100 个 timestamps。

### 1.2 市场微观结构规则

1. 仅最优价可成交，bot 不会越过最优价成交次优价。
2. 无排队。己方订单与市场最优订单同价时，对方优先。
3. 对方行为与市场状态无关，限价单成交率低，可大胆挂单。

### 1.3 策略常量

| 常量 | 值 |
|---|---|
| TARGET | "ASH_COATED_OSMIUM" |
| Q_MAX | 80 |
| TICK | 1 |
| μ | 10000 |
| σ_p | 4.7227 |

---

## 2. 模块复用

以下模块**完全沿用原型策略**，不做修改：

- `outer_fair` 计算（含大单观测、缓存、兜底逻辑）
- `inner_fair` 计算（含 offset 候选、冲突检测、量化到 0.5）
- `memory` 持久化机制

v2 新增逻辑全部构建在 `inner_fair` 之上。

---

## 3. Δq 生成模块

### 3.1 市场位置状态 z

```
z = (inner_fair - 10000) / 4.7227
```

### 3.2 目标仓位 q*(z)

采用线性 + 饱和形式：

```
q*(z) = -Q_MAX · clip(z / z_sat, -1, +1)
```

- `z_sat`：激进取值（待策略撰写者确定具体数值）。
- `q*(z)` 与 z 反号。
- |z| ≥ z_sat 时 q* 饱和至 ±Q_MAX。
- q*(0) = 0。

### 3.3 仓位缺口 Δq

```
Δq = q*(z) - q
```

其中 `q = state.position.get(TARGET, 0)`。

Δq 是 v2 全部下单行为的唯一驱动变量。

---

## 4. 内外层划分

以距 inner_fair 的距离为界：

| 层级 | 判定条件 |
|---|---|
| 外层 | |px - inner_fair| ≥ 5 |
| 内层 | |px - inner_fair| < 5 |

挂单价格永远不得穿越 inner_fair：
- 买单：px < inner_fair
- 卖单：px > inner_fair

---

## 5. Δq 五档分级

根据 Δq 相对 Q_MAX 的比例划分五档（档位边界由策略撰写者确定）：

| 档位 | Δq 特征 |
|---|---|
| 极端多头档 | Δq 大幅为正 |
| 非极端多头档 | Δq 正但不极端 |
| 目标仓位档 | Δq ≈ 0 |
| 非极端空头档 | Δq 负但不极端 |
| 极端空头档 | Δq 大幅为负 |

---

## 6. 五档下单行为表

| 档位 | 外层 | 内层 |
|---|---|---|
| 极端多头档 | 双向挂限价单 | 仅单向买入限价单 + 单向 take 非理性卖单 |
| 非极端多头档 | 双向挂限价单 | 不挂限价单，仅单向 take 非理性卖单 |
| 目标仓位档 | 双向挂限价单 | 双向挂限价单 + 双向 take（与原型策略一致） |
| 非极端空头档 | 双向挂限价单 | 不挂限价单，仅单向 take 非理性买单 |
| 极端空头档 | 双向挂限价单 | 仅单向卖出限价单 + 单向 take 非理性买单 |

**硬性规则**：
- 外层永远双向挂限价单（跨全部五档）。
- 内层行为随档位变化。
- 所有下单量受 Δq 约束（见 §9）。

---

## 7. 外层挂单逻辑

外层双向挂单。每一侧（买 / 卖）独立按以下流程执行：

**Step 1**：检查市场内层是否存在该方向限价单。
- 若存在 → **不挂单**，终止。

**Step 2**：若内层无订单，检查外层是否存在该方向订单。
- 若外层有订单 → 挂在**比市场现存最优价更优 1 tick** 的位置，使用最大可用容量。

**Step 3**：若外层也无订单。
- 查找**上一个状态**的外层挂单位置。
- 若存在 → 在该位置挂单。
- 若不存在 → **不挂单**。

**记忆要求**：每个状态需持久化外层买侧与外层卖侧的上一次挂单价格至 `memory`。

---

## 8. 内层挂单逻辑

仅在当前档位允许内层挂单时执行（见 §6）。每一侧独立按以下流程执行：

**Step 1**：检查内层是否存在该方向订单。
- 大于 inner_fair 处无内层卖单 → 卖方向不挂单。
- 小于 inner_fair 处无内层买单 → 买方向不挂单。

**Step 2**：不得穿越 inner_fair。
- 买单 px < inner_fair。
- 卖单 px > inner_fair。

**Step 3**：按 §6 档位要求挂单，挂单量受 Δq 约束。

---

## 9. 下单量约束

### 9.1 朝向 q* 方向的下单（即与 Δq 符号一致的方向）

- 以 |Δq| 为基准分配。
- 不超过硬容量上限：
  - 买方向容量：`Q_MAX - q`
  - 卖方向容量：`Q_MAX + q`

### 9.2 背离 q* 方向的下单

- 仅在"目标仓位档"的内外层挂单、以及全部档位的外层挂单中允许出现。
- 量由基础做市量决定（具体值由策略撰写者确定）。
- 不超过硬容量上限。

### 9.3 Take 订单

- Take 订单量不超过所 take 价位上对方订单的可成交量。
- 同时受 9.1 / 9.2 的容量约束。

---

## 10. 执行顺序

每个 tick 按以下顺序执行：

1. 加载 `memory`。
2. 计算 `outer_fair`、`inner_fair`（沿用原型策略）。
3. 若 `inner_fair` 为 None → 返回空订单，更新 memory。
4. 计算 z、q*(z)、Δq。
5. 判定 Δq 档位。
6. 生成 take 订单（按档位与容量约束）。
7. 更新 `q` 为 take 后的仓位。
8. 生成外层挂单（§7）。
9. 生成内层挂单（按档位，§8）。
10. 持久化 memory（含外层上一次挂单价格、inner_fair 等）。
11. 返回订单列表。
