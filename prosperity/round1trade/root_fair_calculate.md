

# INTARIAN_PEPPER_ROOT: 核心定价模型规格说明

## 1. 核心数学模型
`fair_price` 为严格的一次函数，最终结果向下取整：
$$FairPrice_t = \text{int}(\mu \cdot t + b_t)$$
* **$\mu$**: 固定漂移常数，$\mu = 0.001$。
* **$t$**: 当前时间戳。
* **$b_t$**: 当前时间戳 $t$ 下的真实基准值。

## 2. 初始基准值 ($b_0$) 锚定
仅在系统启动的首个时间戳 $t_0$ 触发。寻找目标值 $b_{target}$（**100** 的整数倍）：

* **双边订单存在**：计算 $mid = \frac{bid + ask}{2}$，找距 $mid$ 最近的 $b_{target}$。
    * 若 $|mid - b_{target}| \le 5$，则 $b_0 = b_{target}$。
    * 若 $|mid - b_{target}| > 5$，放弃锚定，直接进入滚动计算（见第 3 节）。
* **仅存在 Bid**：在 `(bid, bid + 10)` 找 $b_{target}$。若有则 $b_0 = b_{target}$；若无，赋临时初值 $b_0 = bid + 8$，进入滚动计算。
* **仅存在 Ask**：在 `(ask - 10, ask)` 找 $b_{target}$。若有则 $b_0 = b_{target}$；若无，赋临时初值 $b_0 = ask - 8$，进入滚动计算。

## 3. 基准值 ($b_t$) 滚动计算
**触发**：初始锚定未能成功匹配 **100** 的整数倍时启动。

**1. 数据清洗（每 Tick 执行）**：
* **过滤**：无条件剔除 `size < 15` 的订单。
* **前向填充**：单侧缺失有效订单时，用该侧上一次有效价格填充。
* **计算 $mid_t$**：
    * 双边有效：$mid_t = \frac{valid\_bid + valid\_ask}{2}$
    * 仅 Bid 有效：$mid_t = bid + 8$
    * 仅 Ask 有效：$mid_t = ask - 8$

**2. 无上限 CMA 滚动更新公式**：
设 $N$ 为滚动期间的有效累积 Tick 数（从 1 递增），维护上一时刻的 $b_{t-1}$：
$$b_t = b_{t-1} + \frac{(mid_t - \mu \cdot t) - b_{t-1}}{N}$$

## 4. 备用安全监控模型 (Fallback OLS)
与主模型完全解耦，后台实时运行，负责风控熔断。

**1. 全量 OLS 线性回归**：
在无上限滚动窗口内，收集所有清洗后（剔除 `size < 15` 并完成前向填充）的 $(t_i, mid_i)$ 序列对。
通过最小二乘法实时拟合：
$$mid = \mu_{monitor} \cdot t + b_{monitor}$$

**2. 冷启动预热豁免期**：
为防止开局样本极小时 OLS 参数乱飘导致误报，备用模型触发警报前必须**同时满足**：
* **最小样本量**：$N_{valid} \ge 50$
* **最小时序跨度**：$t - t_0 \ge 5000$ (假设 $t$ 为毫秒级)
*未满足条件时，仅默默更新回归参数，强制拦截所有告警。*

**3. 双重警报触发规则**：
跨过预热期后，实时比对主模型（$\mu = 0.001$, $b_t$）与监控模型参数。若触发以下任一条件，将放弃主模型采用备用模型。
* **基准偏离**：$$|b_t - b_{monitor}| \ge 2$$
* **漂移率偏离 (偏差 > 20%)**：$$\left| \frac{\mu_{monitor} - 0.001}{0.001} \right| > 0.2$$
